# Security Policy

## Supported Versions

Only the latest version of NoiceResume (the `main` branch) receives security updates. We do not maintain backport support for older versions.

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in NoiceResume, please report it privately before disclosing it publicly.

**How to report:**

1. **GitHub Security Tab** (preferred): Go to https://github.com/lovekatoch/noice-resume/security/advisories/new
2. **Email:** If you cannot use the Security tab, email the maintainer directly. The email address is available from the commit history on GitHub.

Do NOT report security vulnerabilities via public GitHub issues, Discussions, or Discord.

### What to Include

- A clear description of the vulnerability
- Steps to reproduce (proof of concept is ideal)
- Affected version(s)
- Any potential impact you've identified

### Response Commitment

| Step | Timeframe |
|------|-----------|
| Acknowledgment of receipt | Within 48 hours |
| Initial triage and severity assessment | Within 7 days |
| Fix development and release | Depends on severity, communicated during triage |

We will keep you informed of progress throughout the disclosure process.

## Disclosure Policy

We follow a coordinated disclosure process:

1. Report received and acknowledged.
2. We investigate, fix, and prepare a release.
3. We notify you when the fix is ready.
4. Once the fix is released, we grant permission for public disclosure.

We strive to resolve critical issues within 14 days of confirmation.

## Scope

This security policy covers the NoiceResume application code hosted in the [noice-resume](https://github.com/lovekatoch/noice-resume) repository and its published deployment at [noiceresume.pages.dev](https://noiceresume.pages.dev). Third-party dependencies should be reported to their respective maintainers.

## Recognition

We maintain a private list of security researchers who have responsibly disclosed vulnerabilities. With your permission, we will credit you in release notes for verified reports.
